# Project Status (Kafka MCP Ops Console)

## Summary
POC is live and functional with a Docker‑based 3‑broker KRaft Kafka cluster, TLS enabled, MCP gateway, and a Web UI that integrates with LM Studio. Read/write tools are separated, with approval required for write tools. The UI is polished for demo and includes tool list, timeline, approvals, and error explanation/search.

## Running Components
- Kafka: 3 brokers in Docker Compose (TLS enabled)
- MCP Gateway: `mcp/server.py`
- Web UI: `webui/server.py`
- LM Studio API: `http://127.0.0.1:1234`

## Start Commands
```sh
# Terminal 1 (MCP)
python3 mcp/server.py

# Terminal 2 (Web UI)
export LMSTUDIO_API=http://127.0.0.1:1234/v1/chat/completions
export MCP_API=http://127.0.0.1:8088
export MCP_APPROVED_BY=operator
export LMSTUDIO_MODEL=<your_model_name>

python3 webui/server.py
```
Open: http://127.0.0.1:8090

## TLS / Certs
- Per-broker certs under `docker/certs/kafka1`, `kafka2`, `kafka3`
- Generated by `scripts/gen-certs.sh`

## Known Issues / Notes
- LM Studio model name must be set via `LMSTUDIO_MODEL` (use `/v1/models` to confirm).
- Broker log tools read from Docker logs for POC; VM/bare‑metal path support is future work.

## Latest Features
- Web UI: tool list with friendly names, timeline (last 5), approvals, health snapshot, error explanation, web search
- Tools added: `kafka_logs_tail_all`, `consumer_lag_snapshot`, `urp_report`, `disk_health`, `broker_status`
- Error explanation auto‑response for “clarify the errors”
